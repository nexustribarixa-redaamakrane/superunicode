# OPENWINDOWS ARCHITECTURE MEMORY: SUCS/EXTSUCS & SUTF/EXTSUTF

> **SYSTEM ARCHITECTURE & TERMINOLOGY SPECIFICATION**
> This artifact stores the complete, authoritative state of SuperUnicode character encodings, SUTF text formatting transports, and the OpenWindows Kernel Mode-Switching Subsystem for direct injection into the future OpenWindows Kernel (`OpenWindows-Kernel`).

---

## 1. System Architecture & Terminology Definition

### A. SuperUnicode (SUCS) and SuperUnicode Extended (ExtSUCS)
- **Classification:** Strictly **CHARACTER ENCODINGS**.
- **Definition:** Defines abstract codepoint numerical address spaces:
  - Base SUCS: 0x00000000 to 0x7FFFFFFF (31-bit bounded).
  - ExtSUCS: 0 -> infinity (unbounded, conceptually infinite; currently implemented via 64-bit `sucs_ex_char_t` container).
- **Rule:** Never refer to SUCS or ExtSUCS as "text engines," "software systems," or "drivers." They are data specifications and bit-level character maps.

### B. SUTF and extSUTF (SUTF-2, SUTF-4, SUTF-8, SUTF-16, SUTF-32, SUTF-64, SUTF-128, SUTF-256, SUTF-512, SUTF-N, vSUTF, e-SUTF)
- **Classification:** Strictly **TEXT FORMATTING AND SERIALIZATION TRANSPORTS**.
- **Definition:** Defines physical byte-packing, bit-alignment, memory layouts, and stream framing rules for storing and transmitting SUCS and ExtSUCS codepoints across memory, SIMD vector registers, IPC channels, and low-level hardware buses.

## 2. Kernel Mode-Switching Subsystem (`sucs_mode.h` / `sucs_mode.c`)

Controls system transitions between Base Mode and Extended Mode:
- **`SUCS_MODE_BASE` (0):** Active Base SUCS 31-bit character encoding & Base SUTF text formatting transports (SUTF-8/16/4/2).
- **`SUCS_MODE_EXTENDED` (1):** Active ExtSUCS 64-bit unbounded character encoding & extSUTF text formatting transports (SUTF-32/64/128/256/512/N, vSUTF, e-SUTF).
- **Restart Sentinel & Commit Rules:**
  - `sucs_request_mode_switch(new_mode)` stages `pending_mode` and sets `reboot_required = true`. Returns `SUCS_SWITCH_REBOOT_REQUIRED`. Active mode is NOT altered immediately.
  - Mandatory system restart is required.
  - `sucs_commit_mode_on_boot(&cfg)` runs during early kernel boot initialization to commit `pending_mode` to `active_mode`, clear `reboot_required`, and increment `mode_change_count`.

## 4. Character Encodings vs. Serialization Transports Matrix

| Layer | Name | Type / Width | Boundary & Validation Rules |
| :--- | :--- | :--- | :--- |
| **Encoding** | **Base SUCS** | 31-Bit (`sucs_char_t`) | `0x00000000`–`0x7FFFFFFF`. Sentinel `0x7FFFFFFF`, Trap Range `0x7FFFFFF0`–`0x7FFFFFFE`. |
| **Encoding** | **ExtSUCS** | Unbounded 0 -> infinity (`sucs_ex_char_t`) | No upper limit or in-band sentinel. Full `uint64_t` range valid except inherited Trap Range `0x7FFFFFF0`–`0x7FFFFFFE`. Out-of-band error handling. |
| **Transport** | **SUTF-8** | 1 to 6 Bytes | Stream framing for Base SUCS codepoints up to 31-bit. |
| **Transport** | **SUTF-16** | 1 to 2 16-Bit Words | `0x0000`–`0xFFFF` 1-word direct (`0xD800`–`0xDFFF` valid PUA); `0x10000`–`0x7FFFFFFF` 2-word (`0x8000 \|` prefix). |
| **Transport** | **SUTF-4** | 4-Bit Hex Nibbles | 8 nibbles (4 bytes) per codepoint for console/bus dumps. |
| **Transport** | **SUTF-2** | 2-Bit Symbol Frames | 16 2-bit frames (4 bytes) per codepoint for IPC channels. |
| **Transport** | **SUTF-32/64** | 4B / 8B Aligned | Fixed-width vector slots for SIMD / AI tensor alignment. |
| **Transport** | **SUTF-128/256/512** | 16B / 32B / 64B Aligned | Zero-padded big-endian SIMD vector register slots (SSE/AVX). |
| **Transport** | **vSUTF** | Variable Multi-Byte | SUTF-8 fast-path for Base SUCS + `0xFE` 9-byte header for 64-bit range (`0xFF` reserved). |
| **Transport** | **e-SUTF** | 6-Byte Page IPC Frame | Hypervisor host/guest page-mapped (4096 codepoint pages) virtual coordinate translation. |

## 5. Kernel & Host Isolation Constraints

1. **Zero Standard Library Dependencies:** All library code under `superunicode/`, `sutf/`, and `superunicode_extended/` compiles strictly with `-std=c99 -nostdlib -ffreestanding`. Zero dynamic memory allocations (`malloc` prohibited).
2. **Forwarding Header Interoperability:** `sutf/include/sutf.h` and `sutf/include/sucs_types.h` contain conditional forwarding headers for `extsucs_types.h`, `extsutf_fixed.h`, `vsutf.h`, and `esutf.h`.
3. **Unified CMake Build System:** All sub-projects are built via CMake (`cmake -B build -G "MinGW Makefiles"`), generating static library targets `libsutf.a` and `libsuperunicode_extended.a`.
4. **OpenWindows Kernel Injection:** When initializing the OpenWindows Kernel project, retrieve this `memory.md` artifact to instantly restore state and inject `libsutf.a`, `libsuperunicode_extended.a`, and `superunicode` into the kernel source tree.
